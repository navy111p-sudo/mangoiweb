# MANGO AI 모바일 메인 — UI 개선 가이드 (2026-06-18)

대상 파일: `cloudflare-deploy/public/index.html`
적용 방식: 파일 맨 끝 `</body>` 직전에 `<style id="ui-polish-2026">` 오버라이드 블록 1개를 추가(가산식). 기존 코드는 삭제하지 않아 안전하며, DOM 마지막에 위치해 우선순위가 가장 높습니다.

---

## ① 메인 타이틀 가독성 (`.ht-glass`)

**왜:** 기존 타이틀은 글자 안이 완전 투명(`-webkit-text-fill-color: transparent`)이라 별·우주 배경과 겹치면 외곽선만 남아 묻혔습니다.

**무엇을:** 네온 외곽선 + 색상 순환 애니메이션은 그대로 두고, 글자 안을 아주 옅게(흰색 12%) 채우고 뒤에 청록 글로우 + 어두운 대비 그림자를 깔았습니다. `paint-order: stroke fill`로 외곽선이 채움 위에 그려져 선이 또렷하게 유지됩니다.

```css
.ht-glass {
  -webkit-text-fill-color: rgba(255,255,255,0.12) !important;
  color: rgba(255,255,255,0.12) !important;
  text-shadow:
    0 0 16px rgba(94,234,212,0.45),
    0 2px 12px rgba(0,0,0,0.60),
    0 0 2px rgba(0,0,0,0.55) !important;
  paint-order: stroke fill;
}
```

> 조절 팁: 더 진하게 원하면 `0.12` → `0.18~0.22`. 완전 외곽선만 원하면 그림자만 남기고 fill을 `0.06`으로.

## ② 곡률(border-radius) 통일

**왜:** 검색창은 18px, CTA 버튼은 데스크톱 18px이지만 모바일(≤640px)에서 14px로 줄어 불일치가 보였습니다. 내부 마이크 버튼은 12px이었습니다.

**무엇을:** 바깥 컨테이너(검색창·결제·수업입장)는 18px로, 내부 액션 버튼(마이크·검색)은 14px로 2단계 스케일을 정리했습니다.

```css
.ai-search-box { border-radius: 18px !important; }
.ai-search-mic, .ai-search-go { border-radius: 14px !important; }
.ai-cta-row > * { border-radius: 18px !important; }
.ai-cta-row > *::before { border-radius: 18px 18px 0 0 !important; }
@media (max-width: 640px) { .ai-cta-row > * { border-radius: 18px !important; } }
```

## ③ 좌측 '열림' 탭 여백/위치

**왜:** 탭(`#mg-drawer-tab`)이 `right:-46px; top:50%`라 화면 왼쪽 끝에 거의 붙고, 가운데 정렬된 검색창과 같은 높이라 겹쳐 보였습니다.

**무엇을:** `right`를 -54px로 키워 왼쪽 끝에서 약 10px 띄우고, `top`을 62%로 내려 검색창 라인을 비켜가게 했습니다. 곡률도 16px로 톤을 맞췄습니다.

```css
#mg-drawer-tab {
  right: -54px !important;
  top: 62% !important;
  border-radius: 0 16px 16px 0 !important;
}
@media (max-width: 640px) { #mg-drawer-tab { top: 64% !important; } }
```

> 참고: 탭은 좌측 드로어(`#mg-drawer`, 화면 밖) 기준 절대위치라, `right` 음수값을 키울수록 화면 안쪽(오른쪽)으로 들어와 왼쪽 여백이 생깁니다.

## ④ 하단 'Mangoi 앱 추가' 배너 겹침

**왜:** 배너(`#pwa-install-bar`)가 `left:12; right:12; bottom:14`로 화면 하단 전체에 깔려, 좌하단 새로고침 계열 FAB·우하단 '상담' FAB와 같은 높이에서 겹쳤습니다. 사용자 눈에는 새로고침 아이콘이 배너 안 캐릭터 아이콘 위에 올라온 것처럼 보였습니다.

**무엇을:** 배너를 코너 FAB(약 bottom:14~16px) 위로 올려(bottom 84~92px) 수직으로 분리했습니다. FAB 자체는 건드리지 않아 동작 영향 없음.

```css
#pwa-install-bar {
  left: 12px !important; right: 12px !important;
  bottom: 92px !important;
}
@media (max-width: 600px) { #pwa-install-bar { bottom: 84px !important; } }
```

## 테스트 & 배포

1. **로컬 점검**: `cloudflare-deploy\test.bat` 더블클릭 → `tests\check-site.js` 가 PASS인지 확인 (이번 변경은 요소를 추가만 했으므로 기존 점검 항목에 영향 없음).
2. **육안 확인**: 모바일 폭(≤640px)에서 ①타이틀 또렷함 ②검색창·버튼 곡률 동일 ③'열림' 탭이 왼쪽 끝/검색창과 안 겹침 ④앱 배너가 상담·새로고침 버튼 위로 떠 겹치지 않음.
3. **배포**: `mangoiweb(last)\deploy.bat` 더블클릭 → 기본(`webrtc-unified-platform`) + 운영(`-prod`) 두 환경 모두 자동 배포 + git push. (실제 사용자 화면은 `-prod` 도메인이므로 둘 다 배포되어야 반영됨.)

> 되돌리기: `index.html` 끝의 `<style id="ui-polish-2026">` … `</style>` 블록만 삭제하면 원상복구됩니다.
