# 망고아이(Mangoi) 안전 배포 — AI 실행용 작업지시 프롬프트

> 아래 내용을 그대로 복사해 AI 에이전트(또는 개발자)에게 작업 지시로 전달하세요.

---

## 역할 / 목표

당신은 망고아이의 배포 담당 엔지니어다. **현재 운영 중인 공식 사이트(`www.mangoi.co.kr`)는 절대 건드리지 않은 채**, 새 WebRTC 화상통화 서비스를 임시 서브도메인에서 테스트하고, 검증이 끝나면 정식 오픈한다.

- 운영 도메인 호스팅: **카페24** (DNS 관리)
- 신규 서비스 호스팅: **Cloudflare Workers** (`webrtc-unified-platform.navy111p...`)
- 테스트용 임시 주소: **`test.mangoi.co.kr`** (서브도메인)

핵심 원칙: 운영 사이트 무중단, 테스트는 임시 서브도메인에서만, 단계별로 검증 후 진행.

---

## 1단계 — 웹 테스트 환경 구축 (`test.mangoi.co.kr`)

목표: 운영 도메인은 그대로 두고, 나만 접근하는 테스트 서브도메인을 만든다. (비용 0원)

1. **카페24 DNS에 서브도메인 추가**
   - 카페24 도메인 관리에서 `test.mangoi.co.kr` 레코드를 신규 추가한다. (기존 `www` 레코드는 수정 금지)

2. **CNAME으로 Cloudflare Workers 연결**
   - `test.mangoi.co.kr` → Cloudflare Workers 주소(`webrtc-unified-platform.navy111p...`)로 향하는 **CNAME** 레코드를 설정한다.
   - Cloudflare 쪽에 해당 커스텀 도메인(custom domain / route)을 등록한다.

3. **SSL(HTTPS) 적용 — 필수**
   - Cloudflare 무료 SSL 인증서를 `test.mangoi.co.kr`에 적용한다.
   - 이유: **스마트폰 카메라/마이크(WebRTC)는 HTTPS가 아니면 동작하지 않는다.**

✅ 완료 조건: 브라우저에서 `https://test.mangoi.co.kr` 접속 시 자물쇠(보안 연결)와 함께 신규 화면이 정상 표시된다.

---

## 2단계 — 앱(WebView) 테스트 (.apk 직접 설치)

목표: 임시 주소를 감싸는 앱 껍데기를 만들고 실제 단말에서 검증한다. (비용 0원)

1. **WebView 앱의 시작 URL 지정**
   - 앱 첫 실행 시 로드 주소를 `https://test.mangoi.co.kr`로 설정한다. (운영 사이트 아님)

2. **카메라/마이크 권한 부여 (WebRTC 핵심)**
   - 앱 매니페스트와 WebView 설정에 카메라·마이크 권한을 선언하고, WebRTC 권한 요청을 허용 처리한다.
   - (Android 예: `CAMERA`, `RECORD_AUDIO` 권한 + `onPermissionRequest` 허용)

3. **.apk 빌드 후 실기기 직접 설치**
   - 스토어 업로드 전, `.apk`를 빌드해 본인 단말에 직접 설치한다.
   - 검증 항목: 화면 반응형(확대/축소) 정상, 카메라 켜짐, 화상통화 연결 정상.

✅ 완료 조건: 실기기에서 앱 실행 → 카메라 켜짐 → 화상통화 정상 동작 확인.

---

## 3단계 — 정식 오픈 (출시)

목표: 테스트 결과가 완벽하면 실제 사용자에게 공개한다.

1. **타겟을 운영 주소로 전환**
   - 앱/서비스가 가리키던 대상을 `test.` → 운영(`www.mangoi.co.kr`) 경로로 전환한다.
   - DNS/Workers route만 전환하고, 운영 트래픽 영향이 없는지 확인한다.

2. **Google Play Store 등록**
   - 구글 개발자 계정 등록비 **약 3.5만 원(최초 1회)** 결제.
   - 빌드한 앱을 업로드 → 심사 후 며칠 내 일반 사용자 설치 가능.

✅ 완료 조건: 운영 주소 정상 서비스 + Play Store에서 앱 설치 가능.

## 안전 가드레일 (반드시 준수)

- 운영 `www.mangoi.co.kr` 레코드/설정은 1·2단계에서 절대 변경하지 않는다.
- 모든 테스트는 `test.mangoi.co.kr`에서만 수행한다.
- 각 단계의 ✅ 완료 조건을 통과해야 다음 단계로 진행한다.
- HTTPS(SSL) 미적용 상태로는 WebRTC 테스트를 진행하지 않는다.
