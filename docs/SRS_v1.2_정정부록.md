## C. ➕ v1.1에 빠진 섹션 (보완 제안)

### C-1. 백업·재해복구 (신규 — "데이터 소실" 우려 대응)
- **Cloudflare D1: 30일 Time Travel(시점 복원) 기본 내장.** 특정 시각으로 DB 복원 가능 → "백업 없어 다 날아간다"는 사실이 아님.
- 소스: GitHub `main`. 녹화·이미지: R2(내구성 보장). Neo4j: 카페24 야간 MERGE(멱등)로 재적재 가능.
- 권장 추가: 정기 D1 export(`wrangler d1 export`) 스냅샷을 주기 저장.

### C-2. 테스트·품질 게이트 (신규)
- 현재: 회귀 하니스(화상 ICE / 정산 43 / 동기화 43)는 있으나 **lint·자동 테스트·배포 전 검증 부재.**
- 권장: 핵심 API 10~15개 스모크 테스트를 `deploy.ps1`에 "배포 전 검사"로 연결 → "하나 건드리면 다 깨진다" 리스크 완화.

### C-3. 성능 (§6 비기능 보완)
- **D1 24시간 rows_read ≈ 4.8억 건** 관측(질의 13,809건 대비 과다) → 인덱스 없는 풀스캔 의심. 상위 질의 프로파일링·인덱스 점검 필요.

### C-4. 학부모 화면 인증 공백 (§3.1 보완)
- `parent.html`에 로그인/토큰 개념이 없음 → 자녀 성장분석(`/api/admin/ai-analyze/student`)이 무인증 노출(현재 유일한 공개 예외).
- 권장: 학부모 토큰 인증(로그인) 도입 후 이 엔드포인트를 잠글 것.

### C-5. 유지보수 리스크 승격 (§7.5 → 1급 로드맵)
- `admin.html` 4만 줄 / `index.html` 2.6만 줄 / `api-mango.ts` 1.2만 줄 모놀리스가 최대 유지보수 리스크.
- 대응: 재작성이 아니라 **점진 분리**(백엔드 도메인별 → 프런트 스크립트 추출). 단계·원칙은 [REFACTOR_PLAN.md](REFACTOR_PLAN.md).

## D. 🔴 v1.1이 옳게 지적한, 아직 남은 진짜 위험 (조기 처리 권장)

| P | 항목 | 내용·대응 |
|---|---|---|
| **P1** | **Neo4j 8880 전세계 개방 + 평문 HTTP** (v1.1 §7.4) | 학생 PII가 암호화 없이 열린 포트로 접근 가능. **재개발이 아니라** Cloudflare Tunnel / nginx TLS / iptables(CF IP 대역 제한)로 잠그면 됨. |
| P1 | `ROOM_JWT_SECRET` 미설정 | 공개 유추 가능한 폴백 사용 중. `wrangler secret put`로 기본+prod 설정 필요(값은 관리자가 지정). |
| P2 | 데모 계정 시드 미포함 | wondang/student가 DB 리셋 시 소실 → 시드 스크립트에 영구화. |
| P2 | admin ivory 테마 가독성(~670곳) | 밝은 배경-밝은 글자. 다크 테마는 완료. |

## E. 협상용 한 줄 정리

> v1.1 SRS는 정확한 문서이고, 시스템은 이미 27페이지로 정밀 문서화돼 있다. 문서가 지적한 **최우선 보안 결함(관리자 무인증 노출)은 이미 근본 수정**했으며, "백업 없음·WebSocket 재작성 필요"는 사실과 다르다. 남은 진짜 항목(Neo4j 포트 잠금, 시크릿 설정, 모놀리스 점진 분리)은 **전면 재개발 없이** 처리 가능하다.
