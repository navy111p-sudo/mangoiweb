# Git 워크플로 한눈에 보기 — 파이썬 파일로 따라하기

앞서 설명해 드린 Git 용어들이 실제로 어떻게 작동하는지, 아주 간단한 **파이썬 파일 하나를 만들고 저장하는 과정**을 예시로 보여드릴게요.
컴퓨터에서 메모장을 켜고 글을 써서 저장하는 것과 똑같답니다!

## 1. Git 실전 활용 예시 과정

우리가 만약 메인 화면을 보여주는 `main.py`라는 파이썬 파일을 새로 만들고 깃허브까지 올린다면, 아래와 같은 순서로 일이 진행돼요.

```text
[1단계: 내 책상 위] 워킹 디렉토리 (Working Directory)
- 컴퓨터 폴더 안에 'main.py' 파일을 새로 만들고 아래 코드를 작성합니다.
  print("안녕하세요! 망고이입니다.")

       ▼ (Git Add 명령어를 실행합니다)

[2단계: 서류 봉투 안] 스테이징 에이전트 (Staging Area)
- "이 'main.py' 파일을 다음 버전에 저장할 후보로 지정할게!" 하고 봉투에 넣은 상태입니다.

       ▼ (Git Commit 명령어를 실행합니다)

[3단계: 내 방 책장] 로컬 레포지토리 (Local Repository)
- "첫 번째 메인 화면 코드 완성!"이라는 메모(커밋 메시지)와 함께 내 컴퓨터에 안전하게 한 버전으로 저장합니다.

       ▼ (Git Push 명령어를 실행합니다)

[4단계: 인터넷 도서관] 원격 레포지토리 (Remote Repository / GitHub)
- 내 컴퓨터 책장에 저장된 버전을 인터넷(깃허브)에 그대로 올려서 백업하고 공유합니다.
```

## 2. 컴퓨터가 이해하는 실제 명령어 예시

실제 개발자들이 컴퓨터 검은색 화면(터미널)에 입력하는 명령어는 다음과 같아요. 흐름만 가볍게 눈으로 보셔도 충분해요!

```bash
# 1. 'main.py'라는 파일을 만들고 코드를 작성합니다. (Working Directory 상태)

# 2. 파일을 서류 봉투에 넣습니다.
git add main.py

# 3. 내 컴퓨터 책장에 메모와 함께 정식 버전으로 저장합니다.
git commit -m "첫 번째 메인 화면 코드 완성"

# 4. 인터넷(깃허브) 공간으로 안전하게 업로드합니다.
git push origin main
```

## 3. 요약 및 핵심 정리

1. **파일 생성:** 내가 컴퓨터에서 `main.py` 파일을 만들고 수정하는 것은 **워킹 디렉토리** 단계입니다.
2. **저장 준비와 완료:** `git add`로 저장할 파일을 고르고, `git commit`으로 내 컴퓨터에 버전 기록을 남깁니다.
3. **최종 업로드:** 내 컴퓨터에 저장된 기록을 `git push`를 통해 인터넷 세상(깃허브)에 안전하게 전송합니다.
