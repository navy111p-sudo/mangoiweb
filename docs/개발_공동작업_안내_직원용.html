망고아이 개발 공동작업 안내

여러 명이 함께 개발할 때 지켜야 할 방법입니다 · 컴퓨터를 잘 몰라도 따라 할 수 있게 썼습니다

인터넷에 “진짜 원본 폴더”를 하나 두고, 각자 복사본을 받아서 고친 다음, 원본에 다시 올리는 방식입니다.
그 방법의 이름이 깃(Git), 그 인터넷 창고의 이름이 깃허브(GitHub)입니다.

1왜 카톡으로 파일을 주고받으면 안 되나요

작업이 사라집니다두 사람이 같은 파일을 고치면, 나중에 보낸 사람 것만 남습니다
추적이 안 됩니다누가 언제 무엇을 고쳤는지 아무도 모릅니다
되돌릴 수 없습니다잘못 고쳐서 홈페이지가 멈춰도 원래대로 돌릴 방법이 없습니다
최신본을 모릅니다내 폴더가 최신인지 아닌지 알 수 없습니다

깃은 이 네 가지를 전부 해결합니다. 특히 되돌리기가 됩니다. 어제 상태로, 3일 전 상태로 언제든 돌아갈 수 있습니다.

2처음 한 번만 하면 되는 준비

1
깃허브 초대 받기

사장님이 이메일로 초대를 보냅니다. 메일에서 수락 버튼을 누르세요.

2
내 컴퓨터로 프로젝트 받아오기

원하는 폴더에서 아래 명령을 실행합니다.

git clone --depth 50 https://github.com/navy111p-sudo/mangoiweb.git

--depth 50은 “최근 기록만 받기”라는 뜻입니다. 전체를 받으면 1.3GB라 오래 걸립니다. 이렇게 받으면 훨씬 빠릅니다.

3매일 하는 일 — 딱 4단계입니다

1
아침에 최신본 받기

다른 사람이 어제 고쳐놓은 것을 내 컴퓨터로 가져옵니다.

git pull

커피 마시기 전에 이것부터 하세요. 안 받고 작업하면 나중에 충돌이 납니다.

2
내 작업방 만들기

원본을 직접 건드리지 않고 나만의 복사본에서 작업합니다. 여기서 무엇을 망가뜨려도 다른 사람에게 전혀 영향이 없습니다.

git checkout -b 학생목록-정렬

뒤의 이름은 자유입니다. 무슨 작업인지 알아볼 수 있게 적으세요. 예) 로그인-오류수정, 교재화면-정리

3
다 고쳤으면 올리기
git add -A git commit -m "학생 목록을 이름순으로 정렬" git push

따옴표 안에는 무엇을 했는지 한글로 적으면 됩니다. 나중에 “이거 누가 왜 고쳤지?” 할 때 이 문장을 봅니다.

4
사장님 검사받기

깃허브 사이트에 들어가면 “Compare & pull request” 버튼이 보입니다. 누르고 제출하면 사장님 화면에 바뀐 부분만 빨강/초록으로 표시됩니다.

사장님이 승인하면 그때 실제 홈페이지에 반영됩니다.

4내가 맡는 영역

두 사람이 같은 파일을 동시에 고치면 충돌합니다. 그래서 영역을 나눕니다.

✅ 내가 맡는 곳 — 학생 학습 콘텐츠

  • cloudflare-deploy/public/student-game-*.html — 게임들
  • cloudflare-deploy/public/student-games.html — 게임 허브
  • cloudflare-deploy/public/vocab.html — 단어장
  • cloudflare-deploy/public/micro-quiz.html — 쪽지시험
  • cloudflare-deploy/public/review-quiz.html — 복습퀴즈
  • cloudflare-deploy/public/warmup.html — 웜업
  • cloudflare-deploy/public/ai-write.html — AI 영작
  • cloudflare-deploy/public/speech-coach*.html — 발음코치
  • cloudflare-deploy/public/battle-3d.html — 보스전
  • cloudflare-deploy/public/suspect-mystery.html — 추리게임
  • cloudflare-deploy/src/api-games.ts — 학습 백엔드
  • cloudflare-deploy/src/api-points.ts — 포인트

게임은 파일이 하나씩 따로 있어서 서로 안 겹칩니다. 실수해도 그 게임 하나만 영향을 받습니다.

🚫 절대 열지 않는 곳

  • cloudflare-deploy/public/index.html — 홈 + 화상수업 (가장 중요)
  • cloudflare-deploy/public/admin.html — 관리자 화면
  • cloudflare-deploy/src/index.ts — 전체 라우팅 · 로그인 관문
  • cloudflare-deploy/wrangler.toml — 서버 설정
  • deploy.ps1 — 배포 스크립트

여기는 잘못되면 서비스 전체가 멈춥니다. 사장님만 만집니다.

담당 밖의 파일을 고쳐야 하는 일이 생기면, 혼자 고치지 말고 먼저 사장님께 말씀하세요.

5배포하면 파일 53개가 바뀐 것처럼 보입니다

⚠ 놀라지 마세요 — 정상입니다

사장님이 배포하시면, 홈페이지 파일 전부에 “언제 배포했는지” 표시가 새로 박힙니다. 그래서 git pull 하면 53개 파일이 바뀐 것처럼 나옵니다. 실제 내용은 그대로입니다.

만약 충돌이 났는데 그 위치가 아래처럼 생긴 줄이면, 아무거나 골라도 됩니다.

그 외의 충돌은 혼자 풀지 말고 물어보세요.

6꼭 지켜야 할 3가지 약속

★
하나. 배포하지 않습니다 deploy.ps1 파일은 실행 금지입니다. 이 파일을 실행하면 내 컴퓨터에 있는 미완성 파일까지 그대로 실제 홈페이지에 올라갑니다. 학생과 학부모가 바로 보게 됩니다. 배포는 사장님만 합니다.
★
둘. 원본(main)에 직접 올리지 않습니다 반드시 3번의 2단계처럼 내 작업방을 만들어서 작업하세요.
★
셋. 아침에 무조건 git pull 먼저 이것만 지켜도 문제의 절반은 안 생깁니다.

7용어 정리 — 서류함으로 비유하면

git pull서류함에서 최신 서류 꺼내오기
브랜치 (작업방)원본 말고 복사본에 연필로 수정하기
git commit“여기까지 고쳤음” 하고 도장 찍기
git push고친 복사본을 서류함에 제출하기
Pull Request사장님께 “원본에 반영해 주세요” 결재 올리기
머지 (merge)결재가 나서 원본에 실제로 합쳐지는 것
배포 (deploy)합쳐진 내용을 진짜 홈페이지에 올리는 것 (사장님만)

8이럴 땐 어떻게 하나요

git pull 했더니 에러가 납니다

혼자 해결하려고 하지 마세요. 화면을 캡처해서 물어보세요. 억지로 해결하려다 작업을 날리는 경우가 가장 많습니다.

내가 고친 게 홈페이지에 안 보입니다

정상입니다. 사장님이 승인(3번의 4단계)해야 반영됩니다.

실수로 파일을 다 지웠습니다

괜찮습니다. 아직 commit 안 했으면 아래로 되돌아갑니다.

git checkout .
다른 사람과 같은 파일을 고쳤다고 나옵니다 (충돌)

5번에 나온 BUILD 줄이면 아무거나 고르면 됩니다. 그 외에는 혼자 풀지 마세요. 잘못 풀면 남의 작업이 사라집니다.

내 담당이 아닌 파일을 고쳐야 합니다

먼저 사장님께 말씀하세요. 특히 index.html과 admin.html은 파일이 매우 커서, 두 사람이 같이 만지면 나중에 합칠 수가 없습니다.
