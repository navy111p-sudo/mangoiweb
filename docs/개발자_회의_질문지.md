# 개발자 회의 질문지 (화요일 미팅용)

목적: 상대가 **정직한지 가려내고**, **전면 재개발(3천만원)을 유지보수 파트너십으로 협상**하고, 우리를 **보호**하기 위한 질문 모음.
각 질문 아래 🟢(좋은 답) / 🔴(주의 신호)로 답을 바로 판단하세요.

> **⭐ 중요도 표시** — 시간이 없으면 별 많은 것부터 물어보세요.
> - **⭐⭐⭐ = 반드시 물어볼 것** (협상의 핵심 — 이것만은 꼭)
> - **⭐⭐ = 중요** (여유 있으면 물어보기)
> - **⭐ = 참고** (시간 남으면)

---

## 회의 전 마음가짐 (3가지)

1. **그 자리에서 계약·결정하지 않는다.** "검토 후 연락드리겠다"가 기본. 모든 건 **서면(견적서·제안서)**으로 받는다.
2. **우리는 이미 손을 봤다는 걸 안다.** 겁주기에 흔들리지 말 것. (보안 핵심은 이미 고쳤음)
3. **우리가 원하는 그림을 먼저 정해두기**: 전면 재개발 ❌ → "우리가 유지하고, 필요할 때만 돕는 파트너" ✅

---

## A. 그들의 진단이 진짜인지 검증 (정직성 테스트)

**A-1.** ⭐⭐⭐ "재개발이 정말 유일한 방법인가요? 지금 코드를 살려서 **부분적으로 개선**하는 방법은 없나요?"
- 🟢 "부분 개선도 가능하다"고 인정 → 정직함
- 🔴 무조건 "처음부터 다시밖에 없다" → 재개발 팔려는 것

**A-2.** ⭐⭐ "보안이 다 열려 있다고 하셨는데, **구체적으로 어떤 엔드포인트**인가요? 저희는 이미 관리자 API 무인증 노출을 **default-deny로 수정·배포**했습니다. 확인해 보셨나요?"
- 🟢 우리 수정을 확인하고 인정 → 실제로 코드를 봄
- 🔴 얼버무리거나 "그래도 다 다시 해야" → 안 봤거나 과장

**A-3.** ⭐ "백업이 없어 데이터가 날아간다고 하셨는데, Cloudflare D1은 **30일 시점 복원**이 기본입니다. 이건 어떻게 보시나요?"
- 🔴 "그래도 위험하다"만 반복 → 사실관계 약함

**A-4.** ⭐ "WebSocket으로 다시 만들어야 한다고 하셨는데, 화상수업은 **이미 WebSocket(Durable Object)으로 작동 중**입니다. 무엇을 다시 만든다는 뜻인가요?"
- 🔴 명확히 설명 못 함 → 시스템을 깊이 안 봄

---

## B. 재개발한다면 — 범위와 리스크

**B-1.** ⭐⭐ "재개발 시 **정확히 무엇을 다시 만들고**, 지금 있는 기능(359개 API, 게임 7종, AI 웜업·코칭, 정산, 이탈분석 등) 중 **무엇이 그대로 유지되고 무엇이 사라지나요?**"

**B-2.** ⭐ "지금 잘 돌아가는 기능이 재개발 후에도 **100% 동일하게** 작동한다고 보장하나요? **기존 기능이 깨지는 것(회귀)**은 어떻게 막나요?"

**B-3.** ⭐⭐ "실데이터(학생 29,288명, 결제 11,083건/약 10.5억원, 수업 50만 건)를 **어떻게 이관**하나요? 이관 중 **데이터 손실·중단** 위험은?"
- 🔴 이관 계획이 두루뭉술 → 위험

---

## C. 비용·기간·책임 (숫자를 쪼개게 하기)

**C-1.** ⭐⭐⭐ "3천만원이 **정확히 어디에 쓰이나요?** 기획/개발/QA/데이터이관/유지보수로 **항목별 금액**을 나눠주세요."
- 🔴 항목 분리를 거부 → 근거 약한 견적

**C-2.** ⭐ "2.5개월이 **늦어지면** 어떻게 되나요? **지연·하자에 대한 책임**은 계약서에 어떻게 명시되나요?"

**C-3.** ⭐⭐⭐ "완료 후 **유지보수는 포함**인가요, 별도인가요? 별도면 **월 얼마**인가요? 무상 하자보수 기간은?"

**C-4.** ⭐⭐ "대금 지급은 어떻게 되나요? **선금 비율**과 **단계별 지급(마일스톤)**이 가능한가요?"
- 🟢 단계별 지급 OK → 건전. 🔴 선금 과다 요구 → 주의

---

## D. ⭐ 우리가 진짜 원하는 방향 — 유지보수 파트너십

**D-1.** ⭐⭐⭐ "전면 재개발 말고, **우리가 지금 시스템을 유지하면서 필요할 때만 도움받는 계약**도 가능한가요? 그 경우 비용은요?"

**D-2.** ⭐⭐ "**월 정액** 또는 **건별**로 지원하는 방식이 가능한가요? 각각 단가는요?"

**D-3.** ⭐⭐ "**긴급 상황(사이트 다운)** 시 연락하면 **얼마나 빨리** 대응하나요? 대응 시간 보장(SLA)이 있나요?"
- 🟢 유연하게 "가능하다" + 명확한 단가/대응시간 → 좋은 파트너 후보
- 🔴 "그런 건 안 하고 재개발만" → 우리에게 안 맞음

---

## E. 실력·정직성 테스트 (우리가 이미 한 것으로 떠보기)

> 우리가 이미 **직접 고친 것**을 상대가 아는지 보면, 실제로 코드를 봤는지 + 실력이 있는지 한 번에 판단됩니다.

**E-1.** ⭐⭐ "예전에 지적하셨던 **Neo4j 포트(8880) 노출**은 저희가 이미 **암호화 터널로 전환하고 포트를 완전히 닫았습니다.** 이 방식(Cloudflare Tunnel)에 대해 어떻게 보시나요? 더 나은 방법이 있나요?"
- 🟢 방식을 이해하고 평가·보완점 제시 → 실력 있음
- 🔴 무슨 말인지 모르거나 "그래도 다 다시 해야" → 스택 경험 없음/과장

**E-2.** ⭐ "**Cloudflare Workers·Neo4j·카페24(CentOS) 서버** 실무 경험이 있나요? **구체적인 프로젝트**를 말씀해 주세요."
- 🔴 우리 스택 경험이 없음 → 학습비를 우리가 대는 셈

---

## F. 우리를 보호 — 소유권·인수인계·종속 방지

**F-1.** ⭐⭐⭐ "코드 **소유권은 100% 우리 것**이 맞나요? 작업물 전체를 **우리 GitHub에 넘겨주나요?**"

**F-2.** ⭐⭐⭐ "당신들 **없이도** 우리(또는 다른 개발자)가 유지보수할 수 있게 **문서화·인수인계**를 해주나요?"
- 🔴 인수인계를 꺼림 → **의도적 종속(계속 돈 내게)** 위험

**F-3.** ⭐ "당신들만 아는 구조나 특정 유료 서비스에 **종속되게 만들지 않는다**고 보장하나요?"

**F-4.** ⭐ "작업 **진행 상황을 우리가 실시간으로 볼 수 있나요?** (GitHub 접근 권한, 주간 보고 등)"

---

## G. 팀·계약 (신뢰 확인)

**G-1.** ⭐ "실제로 **누가 작업**하나요? 그분 **경력**은? **재하청**을 주나요?"

**G-2.** ⭐⭐ "**사업자등록·세금계산서 발행**이 되나요? **계약서**를 쓰나요?"

**G-3.** ⭐ "**포트폴리오**나 비슷한 규모의 **레퍼런스 고객**을 볼 수 있나요?"

---

## H. ⭐ 서버·옛 홈페이지 (오늘 새로 추가된 안건)

> **쉬운 설명:** 아래 두 가지는 "당장 큰일 나는 건 아니지만 미루면 위험이 쌓이는" 것들입니다. 화요일엔 **견적과 일정만** 받아오면 됩니다.

### H-1. ⭐⭐ 낡은 서버 이사 (CentOS 지원종료)

**상황(쉽게):** 우리 데이터가 사는 건물(서버)의 운영체제(CentOS 7)를 만든 곳이 **"이제 관리 안 합니다"**하고 손을 뗐습니다(지원종료=EOL). 금이 가거나 자물쇠가 고장 나도 **고칠 새 부품(보안 패치)이 안 나옵니다.** 당장 무너지진 않지만 갈수록 위험합니다.

**질문:**
- "서버 OS(CentOS 7)가 지원종료 상태입니다. **최신 서버로 이전하려면 비용과 기간**이 얼마인가요?"
- "이전 중 **서비스 중단**이 있나요? 있다면 **몇 시간**이고, **새벽 시간대**로 잡을 수 있나요?"
- "**지금 급한가요, 몇 달 여유가 있나요?** 우선순위를 어떻게 보시나요?"
- 🟢 중단 최소화 계획 + 단계적 이전 제시 → 신뢰 / 🔴 "그래서 전면 재개발해야" → 안건 바꿔치기

### H-2. ⭐ 옛 PHP 홈페이지 안전검사

**상황(쉽게):** 새 시스템(Cloudflare) 말고 **옛날에 만든 별채(PHP 홈페이지, www.mangoi.co.kr)**도 같은 서버에서 돌고 있습니다. 오늘 우리가 **밖에 나와 있던 중요 서류(DB 덤프·소스코드·결제 로그)를 주워 안에 넣었지만(격리 완료)**, 배관 속(코드 내부)까지는 전문가가 봐야 합니다.

**질문:**
- "이 **옛 PHP 사이트, 아직 실제로 쓰나요? 꺼도 되나요?**" (안 쓰면 끄는 게 가장 안전 — 🟢 "꺼도 된다"면 최선)
- "**사용자 업로드 폴더**(uploads)에 나쁜 파일을 올려서 실행되진 않나요? 업로드 파일 검증·차단이 돼 있나요?"
- "오래된 PHP 코드에 **알려진 보안 구멍(SQL 인젝션 등)** 점검을 맡기면 **비용·기간**이 얼마인가요?"
- "**PHP 버전**은 최신인가요, 지원종료 버전인가요?"
- 🟢 "안 쓰면 끄자" 또는 구체적 점검 계획 → 정직 / 🔴 "그래서 전면 재개발" → 회피

## 회의 마무리 (꼭)

1. "오늘 논의한 걸 **견적서·제안서로 정리해서 서면**으로 보내주세요." (범위·금액·기간·책임·유지보수·소유권 포함)
2. **그 자리에서 사인 X.** "내부 검토 후 연락드리겠습니다."
3. 회의 후: 상대 답변을 저(Claude)에게 그대로 알려주시면, **어디가 합리적이고 어디가 과장인지** 같이 판단하겠습니다.

## 한 장 요약 — 우리 협상 목표

- ❌ 3천만원 전면 재개발
- ✅ "우리가 유지 + 급할 때만 돕는" **파트너십** (월정액/건별)
- ✅ 실력 검증: **우리가 이미 고친 것(Neo4j 포트)**을 아는지로 떠보기
- ✅ 새 안건: **① 서버 이전(견적/일정) ② 옛 PHP 사이트 끄기 or 안전검사**
- ✅ **소유권·인수인계·서면** 확보로 종속 방지

## ⭐⭐⭐ 시간 없으면 이 6개만은 꼭

1. **A-1** — 재개발이 정말 유일한 방법인가? (부분 개선은 없나)
2. **C-1** — 3천만원 항목별로 쪼개서 어디 쓰이나
3. **C-3** — 유지보수 포함인가, 별도면 월 얼마인가
4. **D-1** — 재개발 말고 "유지+필요시 지원" 파트너십 가능한가
5. **F-1** — 코드 소유권 100% 우리 것인가 (GitHub 인계)
6. **F-2** — 당신들 없이도 우리가 유지할 수 있게 인수인계하나
