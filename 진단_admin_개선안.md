# 망고아이 실제 관리자(admin.html) 진단 & 우선순위 개선안

> 대상: `cloudflare-deploy/public/admin.html` (= https://webrtc-unified-platform.navy111p.workers.dev/admin.html 의 실제 소스)
> 원칙: 운영 파일이므로 **깨짐 위험이 낮은 것부터**. 각 항목에 위치·근거·효과·리스크 표기.

## 1. 현재 상태 요약 (있는 그대로)

| 항목 | 현황 |
|---|---|
| 규모 | 단일 HTML **36,100줄** · `<script>` 118개 · 인라인 `style=` **3,476개** · 인라인 `onclick` **495개** |
| 디자인 | 다크 테마 토큰 보유 (`--accent #2563EB`, `--bg-card #1E293B`, `--bg-outer #0F172A`, KPI 그라데이션/글로우) |
| 기능 | **76개 기능 카드** + 역할 게이팅(CARD_POLICY/PERMS) + 스코프 격리(본사/지사/대리점) |
| 차트 | Chart.js **4.4.1 (jsDelivr)** · 캔버스 45곳 — *안정적* |
| AI | 이미 **AI 명령바(Phase 21)** 존재 — 검색창 Enter → `/api/admin/ai-command` |
| 반응형 | `@media` 67개 / 접근성 `aria-*`·`role` **28개**(상호작용 대비 적음) |

장점: 기능 풍부, 차트 CDN은 이미 안정적, AI/토글/스코프 인프라가 이미 있음.
약점: 가독성(텍스트 테이블), 일부 CDN 깨짐, 접근성·유지보수성.

## 2. 우선순위 개선안

### 🟢 P1 — 지금 고쳐야 할 실제 결함 (위험 낮음 · 효과 큼)

**1-1. PDF 내보내기 깨짐 (cdnjs 차단)**
- 위치: `admin.html` line ~28606 (`// 2) PDF — jsPDF + html2canvas 동적 로드`)
- 근거: 이 PC/호스트에서 **cdnjs.cloudflare.com 로드가 차단**됨(보안SW). 현재 jsPDF·html2canvas를 cdnjs에서 받으므로 "즉시 PDF 자동생성" 경로가 **조용히 실패**. (Chart.js는 이미 jsDelivr라 정상)
- 개선: 두 라이브러리 CDN을 **jsDelivr로 교체** — `cdn.jsdelivr.net/npm/jspdf@2.5.1/...`, `cdn.jsdelivr.net/npm/html2canvas@1.4.1/...`
- 효과: PDF 자동 저장 복구. 리스크: 매우 낮음(URL 1~2줄 교체).

**1-2. 로딩/에러 가시성**
- 근거: 실시간 수업 등 비동기 영역이 "로딩 중..." 텍스트만 노출.
- 개선: 스켈레톤 UI + 실패 시 "다시 시도" 버튼(기존 새로고침 버튼 재사용).
- 리스크: 낮음(표시 계층만).

### 🟡 P2 — 가독성·즉시대응 UX (우리가 논의한 항목 · 위험 중간)

**2-1. 실시간 수업 현황: 텍스트 테이블 → 카드 + 즉시대응**
- 위치: `id="card-active-rooms"` (line ~2893) — 현재 `<table>`(방 ID/참가자/PDF/영상/액션)
- 개선: 행을 **카드형**으로, ① AI 이상감지(침묵 20s) 행은 `#EF4444` **테두리 Pulse + 최상단 정렬**, ② 우측에 **[즉시 개입]/[GHOST 참관]** 액션 버튼(기존 ghost/whisper API 연결)
- 효과: "지금 개입할 방"이 한눈에. 리스크: 중간(렌더 함수 1곳 교체 — `active-rooms-table` 채우는 JS).

**2-2. 지표 텍스트 → 클릭 링크(동선 단축)**
- 개선: '마감 임박 N건', '미승인 N건' 등 → hover 밑줄+블루(#3B82F6)+pointer, 클릭 시 해당 카드로 점프(기존 `hubJump`/메뉴 점프 재사용).
- 리스크: 낮음.

**2-3. 매출/지표 차트 커스텀 툴팁**
- 근거: Chart.js가 이미 있으므로 external tooltip만 추가하면 됨.
- 개선: 데이터 포인트 hover 시 "N월 매출: ₩XX.XM" 말풍선(CSS) — 캔버스 기본 툴팁 대체.
- 리스크: 낮음(옵션 추가).

### 🟠 P3 — AI 차별화 강화 (기존 인프라 위에 확장)

**3-1. AI 운영 비서 패널**
- 근거: 이미 AI 명령바(Phase 21)·`/api/admin/ai-command` 존재.
- 개선: 단발 응답 → **대화형 패널**(평가서 초안·이상감지 대응·정산 안내)로 확장. 페르소나·번호요약 프롬프트 적용.
- 리스크: 중간(프론트 패널 + 기존 API 연동).

**3-2. AI 이탈 위험군 강조**
- 근거: `card-retention-risk` 이미 존재.
- 개선: '학생 흐름' 영역에 이탈 위험 N명 요약 + 미니 모달(명단·원인·위험도) 진입점 추가.
- 리스크: 낮음~중간.

### 🔵 P4 — 구조·품질 (장기 · 신중 · 선택)

- **유지보수성**: 인라인 `style` 3,476 → 공통 클래스/토큰화, `onclick` 495 → 이벤트 위임. *대규모·고위험* → 카드 단위로 점진 적용 권장.
- **접근성**: 버튼/토글/모달에 `aria-label`·포커스 트랩·`prefers-reduced-motion` 보강.
- **성능**: 36k줄 단일 파일 → 초기 파싱 부담. 비핵심 카드 지연 렌더(이미 `<details>` 구조라 유리).

## 3. 권장 진행 순서 (안전 우선)

1. **P1-1 PDF CDN 교체**(1줄급, 즉시 효과·무위험) → deploy로 검증
2. **P2-1 실시간 수업 카드화 + 즉시대응**(가장 체감 큰 UX)
3. **P2-2 / P2-3 링크·툴팁**(소규모, 누적 효과)
4. **P3 AI 비서·이탈 위험군**(차별화)
5. **P4 구조 개선**(여유 있을 때 점진적으로)

각 단계는 **하나씩 적용 → deploy.bat 배포 → 확인** 사이클을 권장합니다(운영 파일이라 한 번에 크게 바꾸지 않음). 변경 전 `admin.html` 백업도 함께 만들겠습니다.
